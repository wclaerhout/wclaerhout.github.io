Implementing Gradient Descent from scratch in Julia

A walkthrough of vanilla GD, mini-batch variants and the subtleties of step-size scheduling — all in a single Pluto notebook. No Flux, no Zygote: just arrays and loops.

Why re-implement something so basic?

Every tutorial I found either abstracted the update rule into a library call or was written in Python. Neither helped me build real intuition. So I opened a Pluto notebook and wrote it from scratch — slow, explicit, educational.

The best way to understand an algorithm is to be forced to debug your own implementation of it at 1 am.

The update rule

Gradient descent is deceptively simple. Given a differentiable loss L(θ), we repeatedly move θ a small step in the direction of steepest descent: 

# Vanilla gradient descent in Julia
function gradient_descent(loss, grad, θ₀; η=0.01, iters=500)
    θ = copy(θ₀)
    history = Float64[]

    for t in 1:iters
        ∇L = grad(θ)       # compute gradient
        θ  = θ .- η .* ∇L  # update step
        push!(history, loss(θ))
    end

    return θ, history
end

A few things I like about Julia here: the . broadcasting syntax makes the update feel like the math, and there's no overhead from Python's object model hiding what's actually happening.

Step-size scheduling

Fixed learning rates are rarely optimal. I implemented three schedulers — constant, step decay, and cosine annealing — and compared convergence on a simple quadratic. Cosine annealing consistently won on the smoother problems; step decay was more robust on noisy ones. 

# Cosine annealing schedule
η_cosine(t, T, η_min, η_max) =
    η_min + 0.5 * (η_max - η_min) * (1 + cos(π * t / T))

What's in the notebook

The Pluto notebook includes: interactive sliders for learning rate and batch size, live convergence plots via Plots.jl, a comparison between GD / SGD / mini-batch on a logistic regression problem, and a short section on the intuition behind momentum (but no implementation — that's the next post).
