What is the Cross Entropy Method?

CEM is a gradient-free optimisation technique that treats planning as a distribution-fitting problem. Instead of computing gradients, you maintain a distribution over action sequences, sample trajectories from it, discard all but the elite few, and update the distribution to concentrate around those survivors. Repeat until convergence.

It's embarrassingly simple, which is probably why it keeps showing up inside model-based RL systems like Ha & Schmidhuber's WorldModels and its successors. The planner doesn't need to backpropagate through the world model — it just needs to evaluate trajectories, which is cheap.

The best planning algorithm is often the one that makes the fewest assumptions about the model it's planning inside.

The algorithm in three steps

01
Sample. Draw N trajectories (sequences of 2D steps) from the current belief distribution — initially 𝒩(0, 1) for each x/y component at each of the 5 steps.
02
Evaluate & keep elites. Score each trajectory with the cost function (Euclidean distance to target). Sort by cost, retain the top 10% — the elitists.
03
Refit. Compute the mean and variance of the elite trajectories. These become the new distribution parameters. Go back to step 1.

The setup: get to (3, 4)

The problem is intentionally toy: start at the origin, reach (3, 4) using exactly 5 steps. Each step is a 2D vector drawn from a Gaussian. The cost is just Euclidean distance between where you end up and the target. 

# Cost function — Euclidean distance to target
function cost_func(x, y, x_tar=3, y_tar=4)
    cost = sqrt((x - x_tar)^2 + (y - y_tar)^2)
    return cost
end

# Turing model: sample 5 2D steps from current belief
@model function action_sampling(μ_vec, σ_vec)
    steps = zeros(5)
    for i in 1:length(steps)
        steps[i] ~ MvNormal(μ_vec[i], σ_vec[i])
    end
    x_final = sum([x[1] for x in steps])
    y_final = sum([x[2] for x in steps])
    cost = cost_func(x_final, y_final)
    return (x_final, y_final, cost)
end

Initial belief: all means zero, all variances one. 200 trajectories sampled from the prior. The scatter is wide — most land nowhere near (3, 4).

Fig 1 — 200 naive trajectories sampled from 𝒩(0,1) prior. Wide scatter, few near the target.

Finding elites

From the 200 samples, the 20 with the lowest Euclidean distance to the target are kept. These are the elites — green dots in the plot below. They cluster noticeably closer to (3, 4) than the orange cloud. 

function return_idxs(costs)
    elite_scores = sort(vec(costs))[1:20]       # top 10%
    idxs = zeros(Int, length(elite_scores))
    for (i, score) in enumerate(elite_scores)
        idxs[i] = findfirst(x -> x == score, vec(costs))
    end
    return idxs
end
Fig 2 — Green = elite 10%. Their cluster already sits near (3, 4).

Updating beliefs

The elite trajectories are used to refit the distribution. For each of the 5 steps and each of the 2 dimensions, we compute the sample mean and variance across elites. These become the new μ and σ². 

function update_vars(action_chain, idxs)
    elite_chains = action_chain[:steps][idxs]

    trajectory_array = zeros(Float64, 20, 10)
    for (i, chain) in enumerate(elite_chains)
        trajectory_array[i, :] = reduce(vcat, chain)
    end

    up_means = vec(mean(trajectory_array, dims=1))
    up_vars  = vec(var(trajectory_array,  dims=1))

    updated_means = [up_means[i:i+1] for i in 1:2:10]
    updated_vars  = [up_vars[i:i+1]  for i in 1:2:10]

    return updated_means, updated_vars
end

The notebook plots the refitted Gaussian for each of the 5 x and 5 y step distributions (a 2×5 grid of density curves). After one iteration, the means have shifted substantially toward the values that sum to 3 and 4 respectively, and the variance has tightened.

One iteration later

After a single CEM iteration the new samples cluster dramatically closer to the target. The histogram of Euclidean distances (not shown here, but in the notebook) shifts its mode from ~5 down to ~1–2. A second or third iteration tightens this further.

Fig 4 — After one CEM iteration the entire sample cloud has converged near the target.

Continuation

You repeat the loop — sample, evaluate, keep elites, refit — until you're satisfied with the found policy. In practice two or three iterations on this toy problem brings the mean distance below 0.5. In LeWorldModel the same loop runs inside the latent space of the world model, planning over imagined rollouts rather than ground-truth steps. 

# The full CEM loop
means, vars = initial_belief()

for iter in 1:n_iters
    chain   = sample(action_sampling(means, vars), Prior(), 200)
    results = generated_quantities(action_sampling(means, vars), chain)
    costs   = [r[3] for r in results]
    idxs    = return_idxs(costs)
    means, vars = update_vars(chain, idxs)
end

That's the whole algorithm. What makes CEM attractive for model-based planning is that it requires zero gradient information — the world model only needs to be differentiable with respect to its own training loss, not the planning objective. The planner treats it as a black-box simulator.
